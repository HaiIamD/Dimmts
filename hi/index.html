Virtual Private Database

VPD (Virtual Private Database) suatu fitur yang memiliki fungsi untuk membuat kebijakan keamanan dalam mengontrol akses database pada tingkat baris dan kolom. Virtual Private Database memberlakukan keamanan, ke tingkat granularitas yang baik, langsung pada tabel database, tampilan, atau sinonim dan melampirkan kebijakan keamanan langsung ke objek database ini, dan kebijakan diterapkan secara otomatis setiap kali pengguna mengakses data, tidak ada cara untuk melewati keamanan.

Kebijakan keamanan Oracle VPD menyediakan mekanisme untuk mengamankan data pada tingkat database itu sendiri. Kemampuan untuk mengamankan data pada tingkat objek database granular adalah fitur VPD yang sangat kuat. Secara prinsip,  Oracle Virutal Private Database menambahkan klausa WHERE dinamis ke pernyataan SQL yang dikeluarkan terhadap tabel, tampilan, atau sinonim dari kebijakan keamanan Oracle Virtual Private Database yang diterapkan.

VPD biasa digunakan ketika hak istimewa objek dan peran basis data tidak memadai untuk mencapai persyaratan keamanan. Kebijakan atau protokol berbanding lurus dengan persyaratan keamanan. 

 

VPD memiliki beberapa keuntungan dan kerugian dari berbagai aspek seperti berikut : 

Keuntungan VPD:

Aksesibilitas Lebih Tinggi: Pengguna dapat dengan mudah mengakses data dari mana saja.

 .      Fleksibilitas: Dapat dengan mudah dimodifikasi tanpa merusak aliran kontrol.

.      Tingkat Pemulihan Lebih Tinggi: Data dapat diambil dengan sangat mudah.

        Diamankan Secara Dinamis: Tidak perlu mempertahankan peran        yang kompleks.

Tidak ada pintu belakang: Kebijakan keamanan dilampirkan pada data sehingga tidak boleh melewatinya

Kekurangan VPD:

Keamanan tingkat kolom yang sulit

ID akun Oracle diperlukan untuk menggunakan layanan ini.

Sulit untuk diperiksa

Iplementasi VPD 

Membuat Database terlebih dahulu 

Selanjutnya kita dapat membuat table dan mengisi table tersebut, diaman kita menggunakan metode bistring yang dapat untuk menyimpan banyak label keamanan secara efisien dengan cara Setiap label diberi bit field , dan kemudian operasi bitwise dapat digunakan untuk mendapatkan/mengatur masing-masing label dari bitstring.

        Mulai membuat simpan tingkat akses sebenarnya untuk akun pengguna denga cara membuat table baru lagi

       Membuat fungsi tersimpan untuk mewakili kebijakan keamanan tingkat baris kita . Fungsi mengambil nama pengguna X dan label keamanan Y , dan mengembalikan nilai true jika pengguna diizinkan untuk mengakses label.

      Sekarang kita mulai menguji fungsi yang telah kita buat sebelumnya 

       Sekarang yang perlu kita lakukan ialah menyiapkan beberapa data untuk dilindungi.Pertama, mari buat tabel biasa dengan beberapa data berlabel.

Kedua buat tampilan yang menanyakan tabel yang tidak dilindungi dengan cara yang aman .

       Sekarang yang perlu kita lakukan ialah membuat akun anonim dan berikan akses ke database yang dilindungi .

Sekarang kita dapat masuk sebagai pengguna mana pun ke database ini.

      Sekarang kita bisa melakukan pengujian menggunakan beberapa pengguna yang berbeda
